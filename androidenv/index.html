miniconda

pyenv依赖问题一直很麻烦，可以切到miniconda

  1. 下载安装脚本（建议科学）：
    proxychains wget

https://repo.anaconda.com/miniconda/Miniconda3-lat…

  1. 给安装脚本可执行权限
    chmod +x Miniconda3-latest-Linux-x86_64.sh
  2. 运行安装脚本：
    sh Miniconda3-latest-Linux-x86_64.sh
  3. 安装python3.8.0环境（建议科学）
    proxychains conda create -n py380 python=3.8.0
  4. 切换到python3.8.0
    conda activate py380
  5. 安装frida 12.8.0全家桶（建议科学）
    proxychains pip install frida12.8.0
    proxychains pip install frida-tools5.3.0
    proxychains pip install objection==1.8.4
  6. 为了新建的终端默认的env在py380，可以将下面这句话放在/root/.bashrc的末尾，然后source一下。

conda activate py380

  1. 也可以把这句话放在/root/.bashrc的末尾，source之后只要运行mfrida命令即可切换到py380环境。

alias mfrida=”conda activate py380”

  1. 第七条和第八条二选一即可。开始享受12.8.0全家桶吧~

android 系统刷机

准备

  • 手机: Nexus 6P
  • 系统: google 官方工厂镜像

使用 wget 命令下载目标系统镜像, 下载完成后使用 shasum -a 256 /tmp/hello.txt 进行文件校验;

刷机

fastboot

第一步, 将手机进入 fastboot状态, 操作如下:

  1. 将 USB 断开, 确保手机有 80% 以上的电量;
  2. 将手机完全关机
  3. 同时按住音量向下键和开机键
  4. 手机进入 fastboot状态

如果手机没有解锁, 那么 Device State 显示 Locked; 需要先解锁, 只有 Unlocked 才可以继续刷入;

刷入系统

第二步, 手机用 USB 连接电脑, 运行脚本, 将系统刷入手机

刷机完成后手机可以正常开机;

刷入 twrp

第三步, 下载twrp针对对应机型的镜像, 手机再次进入到 fastboot 模式, 使用 fastboot 命令将镜像刷入;

刷完之后, 手机按两次音量向下键, 选择 Recovery mode, 按电源键进入, 稍等片刻即可进入 twrp 界面;

进入 twrp 界面后, 直接滑开即可, 意味着允许修改系统;

刷入 Magisk

从 Magisk 的 github 中下载最新的卡刷包 magisk.zip

然后使用 adb 命令将卡刷包上传到手机中 adb push magisk.zip /sdcard/

然后使用 twrp 将这个卡刷包安装到手机里, 打开 twrp, 点选 Install, 选择刚刚传入进去的 magisk.zip 文件, 滑动确认安装;

安装完成后, 直接重启即可; Reboot System;

root

  1. 设置 --> 系统 --> 关于手机 --> 版本号, 点击版本号五下, 打开开发者选项, 然后进入 设置 --> 系统 --> 开发者选项;
  2. 打开 USB 调试选项;
  3. USB 连接电脑, 使用 adb 命令连接;
  4. 手机授权点击同意
  5. su
  6. 手机出现 Magisk 的超级用户请求, 点击允许;

charles 抓包

  • http://zhaoxincheng.com/index.php/2020/07/16/安卓7-0及以上抓包证书小技巧/
  1. 将证书安装到 user 下
  2. 用re管理器把/data/misc/user/0/cacerts-added/这个路径下面的文件复制到/system/etc/security/cacerts
