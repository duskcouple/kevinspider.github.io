script标签

发表于2021-06-30|更新于2021-07-29|js基础

<script>元素

<script>属性

将 javascript 插入到 html 中, 主要的方式是使用<script>标签, 该元素有 8 个属性;

  1. async: 可选, 表示应该立即开始下载脚本, 但不能阻塞其他页面的动作, 只对外部文件有效;
  2. charset: 可选, 表示了使用src属性指定的代码的字符集, 这个属性基本不使用, 浏览器也不在乎它的值;
  3. crossorigin: 可选, 配置相关请求的 cors(跨源资源共享), 默认不使用 cors, crossorigin="anonymous"配置文件请求不必设置凭据标志; 当 crossorigin="use-credentials"设置凭据标志, 意味着出站请求会包含凭据标志;
  4. defer: 可选, 表示脚本可以延迟到文档完全被解析和显示之后再执行, 只对外部脚本文件有效;
  5. integrity: 可选, 允许对比收到的资源文件和指定的加密签名, 以此来保证资源的完整性和安全性; 如果收到的资源的签名和这个属性指定的签名不匹配, 则页面会报错, 脚本不会执行; 这个属性可以用于确认内容分发网络 cdn 不会提供恶意的内容;
  6. language: 废弃, 最初用来表示代码块中的脚本语言, 大多数浏览器都会忽略这个属性;
  7. src: 可选, 表示包含要执行的外部代码的路径;
  8. type: 可选, 代替 language; 表示了外部代码脚本语言的内容类型, 按照惯例一般都是text/javascript; 如果这个值为module, 则代码块中的代码会被当成是 ES6 中的模块, 只有这个时候, 代码中才可以出现 import和export关键字;

defer 和 async

defer表示推迟脚本执行, 这个属性只适用于外部的 js 脚本, 表示脚本在执行的时候不会改变页面结构, 脚本会在整个页面都解析完成后再运行, 设置了defer就是告诉浏览器, 立即下载这个脚本但是延迟执行; 虽然 html5 的规范中规定了, 使用了defer属性的 js 脚本应该是按照<script>出现的顺序依次执行, 且必须在 DOMContentLoaded事件之前执行, 但是实际上不一定都遵守这个规范, 所以推荐最好只包含一个带有defer属性的脚本;

async属性表示了当前加载的 js 支持异步, 标记了async之后不会阻塞主进程, 同时也不会按照次序执行;

  1. 第二个脚本可能先于第一个脚本执行
  2. async的作用是告诉浏览器, 不必等待 javascript 脚本下载完毕后再加载页面, 也不需要等该异步脚本下载和执行后再加载其他脚本
  3. 因为异步存在, 所有使用了该属性的 javascript 脚本不应该在加载期间修改 dom 节点;
  4. 并不推荐使用这个属性;

<script>使用方式

在 html 代码中直接嵌入 javascript

  1. 包含在<script>内的代码会被从上到下解释

  2. 在<script>元素中的代码被计算完成之前, 页面的其余内容都不会被加载, 也不会被显示, 这个过程会阻塞

  3. 在使用行内 javascript 的时候, 代码中不能出现字符串 </script>, 因为浏览器在解析脚本的时候会将</script>当做是 javascript 结束的标志; 需要使用转义字符来避免这个问题

引入外部文件

通过引入外部文件中的 javascript 代码, 使用 src属性来指定一个外部的 url 地址;

  1. 和解释行内 javascript 一样, 在解释外部 javascript 文件时, 也会阻塞页面;

  2. 在使用了src属性之后, <script>标签内不能再包含其他的 javascript 代码, 即使包含了也不会被执行;

  3. <script>可以直接引入其他域的 js 代码, src属性可以是一个完整的 url, 而且这个 url 可以执行和 html 不在一个域下面的脚本文件;

  4. 浏览器会按照<script>在页面中出现的顺序依次解释它们, 前提是它们没有包含async和 defer属性; 第二个<script>只能在上一个<script>元素的代码解释完毕之后才能开始解释;

小结

javascript 是通过<script>标签插入到 html 代码中的, 这个元素可以用于把 js 代码嵌入到 html 页面中, 和其他标记混合在一起使用, 也可以用于引入保存在外部文件中的 js;

  • 要包含外部的 js 文件, 必须将 src属性设置为要包含文件的 url 地址, 文件可以和网页在同一个域下也可以不在同一个域下;
  • 所有 <script>元素会按照它们在页面中出现的次序被解释, 在不使用defer和async属性的情况下, 包含在<script>元素中的代码会严格按照出现的次序依次执行;
  • 对不推迟执行的脚本, 浏览器必须解释完位于<script>元素中的代码, 然后才能继续渲染页面的剩余部分; 因此, 通常应该把<script>元素放在页面末尾, 介于主内容以及</body>标签之前;
  • 可以使用 defer 属性把脚本推迟到文档渲染完毕之后再执行, 推迟的脚本原则上按照它们被列出的次序进行执行;
  • 可以使用 async 属性表示脚本不需要等待其他脚本, 同时也不阻塞文档的渲染, 即异步加载; 异步加载不能保证按照它们出现在页面的次序执行;
