准备工作

下载 app: https://pan.baidu.com/s/18_WCIb_KaKCAgxLj_1bLMA 提取码：k1f3

不需要脱壳, 直接使用 jadx 打开即可;

java 层定位

在登录界面, 输入用户名和密码后点击登录, 抓包可以看到 post 请求中有一个参数 s; 每次输入的用户名和密码不同时 s 的值也不相同; 当输入的值固定时, s 每次的结果都相同;

jadx 中直接搜索”s”, 可以发现:

查看 ParamsKey 中 s 属性的调用, 可以定位到 com.weico.international.activity.SinaWeiboUserLoginActivity.doLogin方法:

进入calculateSInJava中, 可以看到最终调用的是com.sina.weibo.security.WeiboSecurityUtils.calculateS方法, 该方法是一个 native 方法; 使用 frida 对该方法进行 hook , 验证静态分析的结果;

第一个参数是 context, 第二个参数是输入的用户名+输入的密码; 第三个参数是定值字符串; 结果和抓包结果对比也一致;

java 层主动调用

native 层分析

对 apk 解包, 找到libutility.so, 拖进 ida 中; 发现直接在 exports 中搜索 calculateS可以看到导出函数, 说明该方法是静态注册;

先将入参JNIEnv 等设置为正确的类型, 读起来还是比较明晰的;

最终结果是来自 v20, 而 v20 其实是由 v12 产生的, v12 是 MDStringOld(v10) 的结果, 而 v10 其实是输入的两个字符串参数拼接的结果; 使用 frida hook 验证猜想;

使用 cyberchef 验证, 确实是标准的 md5 的结果, 接下来就是对 md5 结果的截取的位置调换, 形成最终的s;

unidbg

脚本直接使用 Unidbg 运行会出现报错, 龙哥给出的是 patch 的方式直接跳过判断函数的执行, 设置 r0 寄存器的值为 1; https://armconverter.com/?code=mov%20r0,1, 获得 thumb 的cpcode 为: 4FF00100; 使用 unidbg 设置 patch

或者直接使用 010Editor 打开目标 so, 将原 cpcode FF F7 EB FE修改成4F F0 01 00然后运行, 测试也是可以的;
